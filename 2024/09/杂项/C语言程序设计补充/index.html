对 The C Programming Language 的几点补充

发表于 2024-09-07 更新于 2024-10-07 分类于 杂项

大学军训的最后几天, 我稍微花了点时间看完了 The C Programming Language 一书, 并写下了一些初学者阅读本书的时候可能遇上的问题.

C 语言非常简洁, 相信学会其语法不需要太多时间.

关于 C 语言的标准

本书所定义的是最早的 C 语言标准, 所以与现在的 C 语言略有不同.

  1. 本书中函数返回值可以省略, 所以你在本书会看到 main(), 但是在以后的 C 语言标准中需要使用 int main().
  2. 本书中变量要先声明才能使用, 事实上从 C89 开始就不用了, 可以直接 int flag = 0;
  3. 本书中注释是 /* */, 但是从 C99 开始就可以用 // 来表示单行注释了.

C 语言标准大概有以下几个大版本:

  1. K&R C
  2. ANSI C (C89)
  3. C99: 引入 inline, long long int, _Bool, 支持复数(complex.h)
  4. C11: 引入多线程支持, 增加了对原子操作的支持

关于布尔运算符

与 python 等语言不同, 在 C 语言中, 布尔运算符是 ! && ||, 分别表示 not and or.

个人认为这样是有道理的, 至少这三个符号比起英文单词要明显得多, 可以很快看出表达式的层次.

缩进

另外, 本书中的缩进似乎是 4 个空格, 历史上对于缩进的争论很多, 有人选择 tab(制表符), 有人选择 2 个空格, 也有人选择 8 个空格, 我与本书的选择一致.

列宽

由于早期的计算机终端每行最多显示 80 个字符, 所以每行最多 80 个字符成为了一个代码规范.

当然, 现代显示器支持更大的列宽, 也有开发者选择 120 列宽.

我目前还是选择 80 列宽, 毕竟我的电脑可以显示下 160 列宽, 我的屏幕可以平铺两个界面.

补充一下关于编译参数的知识

基本编译选项:

有几个常用参数

  1. -std=standard 指定标准.
  2. -Wall -Wpedantic -Wextra 启用各种警告.
  3. -g 生成调试信息, 如果需要 debug 请使用这个参数.
  4. -O 使用优化, 一般使用 -O2 (俗称氧气).
  5. -D 定义宏, 比如 -DDEBUG 定义了宏 DEBUG, 可在代码中 #ifdef DEBUG 来进行条件编译.
  6. -U 取消定义宏, 比如 -UDEBUG 取消定义 DEBUG.
  7. -fsanitize=address 启用对内存错误的检测 (比如缓冲区溢出, 数组越界).
  8. -fsanitize=undefined 启用未定义行为的检测.

杂项

  1. stdlib.h 中定义了几个比较有用的函数, 比如 atoi(), 初见这个名称我有些奇怪, 后来感觉也许这指的是 array to integer, 也就是把字符数组转换为整数.
  2. debug 的时候可能需要反复读入输出数据, 有一种比较简单的方式就是 freopen(), 把 stdin, stdout 重定向到文件里.
  3. C 语言有些函数其实是不安全滴, 比如 gets(), 它好像已经被抛弃了.

阅读结束之后

本书介绍的都是 C 语言的基础, 掌握了这些确实是可以写一些小项目了, 但是 C 语言其实还是有很多技巧的, 比如宏, 它其实是一个很复杂的话题, 善用宏可以极大的简化代码复杂度. 另外在代码行数过万行且有多个源代码文件的时候, 学会使用一个现代化的 IDE 就很有必要了, 这可以极大的增加写代码的效率.

最后, 编程并不是看会的, 而是练会的, 只看书或者刷题目远远不够, 还是需要自己写一定量代码才够.
